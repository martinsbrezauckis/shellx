# Polish UI

Make the interface clearer, tighter, and more consistent.

1. Inspect existing tokens, components, icon system, and spacing rules.
2. Pick the main usability problem: hierarchy, density, alignment, overflow,
   contrast, empty state, or responsive fit.
3. Keep the current product style. Do not invent a new palette or card system.
4. Use familiar icons for compact actions; add tooltips for icon-only controls.
5. Verify desktop and narrow/mobile widths. Check that text does not overlap or
   overflow buttons, tabs, cards, or sidebars.
6. For rich visuals, use screenshots to compare before/after.
7. Report the visible changes and any viewport not tested.

Avoid: decorative gradients, nested cards, oversized hero type in app panels,
and visible instructional text that belongs in docs.
