# Changelog

All notable changes to shellX are documented here.

Format: [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
adapted for shellX public releases.

## [0.1.29] — 2026-05-22 (public beta release candidate)

### Fixed — security and release readiness

- **A0 STRUCTURE** Bundled `shellx-host` skill source now lives under
  public `skills/shellx-host/SKILL.md`. The binary include path and
  integration test point at that public source of truth.
- **A1 HIGH** `read_image_as_data_url` lacked the renderer-side
  credential denylist that `read_text_file_for_path` had. Both
  renderer read commands now reject `auth.json`, `vault.*`, `*.token`,
  `.netrc`, `.pgpass`, `.ssh/id_*`, `.aws/credentials`,
  `.password-store/`, `.gnupg/` regardless of scope.
- **A2 MED** Host MCP HTTP write-class tools default-deny on missing
  `MCP-Tab-Id` header (instead of falling back to "default" tab and
  silently bypassing per-tab plan/observe).
- **A3 MED** Vault fallback master key write switched from
  `write+chmod-after` to atomic `OpenOptions::mode(0o600).open()` on
  Unix (no umask window).
- **A4 MED** README claimed `/imagine /research /refactor` slash
  commands — only `/goal` + `/pr` are wired. Trimmed claim and removed
  orphan private skill notes from the release tree.
- **A5 MED** Stale internal-doc refs in `docs/API.md`,
  `ShellxagentTab.tsx`, `RightRail.tsx` rewritten to live docs.
- **A6 MED** Real CI added: `.github/workflows/ci.yml` runs tsc + unit
  tests + cargo fmt + clippy + cargo-audit on every push. `pnpm test`
  / `pnpm lint` / `pnpm typecheck` scripts added. THREAT_MODEL.md
  references the actual workflow.
- **A7 MED** Branch scope-pill fallback `"master"` → `"—"`; autonomy
  chip + command palette unified on Confirm / Auto labels; AboutModal
  deleted, Header brand opens Settings → About (single canonical
  About surface).
- **A8 LOW** Settings → Data "Cache cleanup: Soon" disabled row
  removed. PR-create modal `"- [ ] TODO"` replaced with neutral
  italic description.
- **A9 LOW** Private skill-manifest notes removed from tracking — they
  are not runtime-bundled. The bundled host-skill source now lives under
  `skills/shellx-host/SKILL.md`.
- **A10** `cargo fmt` applied across the workspace.

### Fixed — orchestration and transport

- **B1 CRITICAL** `/autonomy` on a live session can't apply mid-process
  (grok bakes `--always-approve` into argv at spawn). Backend
  already returned `appliesAfterReconnect:true` honestly; UI now
  consumes it and surfaces a chat toast (`⚙ autonomy → <mode> —
  apply on reconnect`) so the user knows to /abort + /connect.
- **B7 LOW** `POST /sessions/:id/archive` returned 500
  "session has no cwd yet" because the route was aliased to the tab-id
  handler. Now resolves the session id → owning tab id via
  `SessionRegistry::find_tab_by_session_id` and 404s honestly when
  no live tab owns the id.
- **B8 MED** SSH connects logged 6× "Failed to spawn MCP server
  'shellx-mp-*'" because shellX injected stdio marketplace entries
  (uvx, npx, cmd.exe …) into the remote grok config. New
  `build_session_new_entries_for_transport()` skips stdio entries on
  WSL/SSH transports; HTTP/SSE entries still inject.
- **B9 HIGH** `/goal` completion validation now rejects false final
  evidence phrases such as "tested via code paths" / "verified by
  inspection", and rejects self-review as a substitute after reviewer
  transport or dispatch failure. This turns silent reviewer-transport
  shrinkage into a hard
  `goal_complete` refusal instead of an accepted completion.
- SSH subagents now request the same reverse host-MCP tunnel as parent
  SSH sessions, so reviewer / implementer agents can reach shellX host
  tools from PC2-style remotes instead of seeing MCP as unavailable.
- SSH session startup now installs the bundled `shellx-host` skill into
  the remote `~/.grok/skills/shellx-host/SKILL.md`, keeping Local, WSL,
  and SSH guidance consistent after reinstall or remote drift.
- `/goal` approval now updates the top-level `goal.md` status to
  `IN_PROGRESS` through shellX's transport-aware writer before
  resuming Grok. This removes the SSH approval deadlock caused by
  native remote file-edit mismatches.
- Past-chat history now persists connection metadata into the session
  JSONL and recovers it on reopen. Closed WSL / SSH sessions show their
  correct transport icon and reconnect through the original saved
  preset instead of silently falling back to Local.
- Voice-chat replies are scoped per tab and generated by shellX from
  the assistant's final text when voice mode is enabled; ordinary
  replies no longer ask Grok to call `voice_tts` directly.

### Fixed — updater and release pipeline

- Header update banner is mounted above the app header and uses the
  Tauri updater plugin to check GitHub Releases `latest.json`, download
  signed update artifacts, install, and relaunch.
- Settings → About now uses the same updater plugin path and exposes
  an `Install & restart` action when a release is available.
- GitHub release workflow builds signed updater artifacts on `v*` tags
  and links release notes to the exact commit SHA rather than assuming
  a `master` default branch.

### Polish — personas + host skill (instruction surface)

- Five subagent personas (`general-purpose`, `explore`, `implementer`,
  `reviewer`, `security-auditor`) rewritten into a compact drop-in
  pack; ~33 % shorter, same operational content. Implementer
  keeps the stack-specific `cargo fmt + clippy + test` cue;
  security-auditor keeps the severity → implementer-severity mapping.
- `skills/shellx-host/SKILL.md` keeps the compact host manifest while
  preserving every load-bearing specific: vault `vault:<key>` prefix
  rule, `grok-shell-host__goal_complete` exact tool name, WSL bridge
  `wsl.exe -d <distro>` spawn pattern, image/video path conventions,
  ACP-mode bundled-skill names that hang on the `task` tool,
  autonomy-gate JSON-RPC error code -32001.
